VoetstAPPen

com.example.voetstappen_app

Fitness tracking and audio-guided workout app. Users can track walks/runs via GPS, listen to audio guides, view routes on maps, and store workout history locally.

Claimed: React Native Actual: Flutter (Dart) Health & Fitness
Critical Framework misrepresentation confirmed
Narola Infotech claims this app was built with React Native, but every dependency in the project is a Flutter/Dart package. The project path uses Flutter's default com/example/voetstappen_app structure. React Native uses completely different packages (react-native-sound, react-native-maps, @react-native-async-storage, etc.).

Key findings

Critical issues found across both audited applications. Click an item to jump to its detail.

Critical Hardcoded credentials in production
QuadMaps ships with 6 hardcoded secrets — including API keys, billing keys, and map tokens — directly in the source code. One MapBox token is already dead (401), meaning it was likely revoked after being exposed, yet remains in the build.
Critical Framework misrepresentation
VoetstAPPen is claimed to be built with React Native but is provably Flutter. Every dependency is a Flutter/Dart package. This either reflects dishonesty in portfolio claims or a fundamental disconnect between sales and engineering.
High Deprecated technology stack
QuadMaps relies on Retrofit 1.x, OkHttp 2.x, RoboSpice, and 51 AsyncTask classes — all deprecated or end-of-life. AsyncTask was deprecated in Android API 30 (2020). The app was last built in July 2025, meaning these deprecated patterns are still actively shipped.
Medium Default project scaffolding left in place
VoetstAPPen still uses the com.example package prefix — the default from flutter create. This is a scaffolding leftover that should be replaced with the client's domain for any production release.
